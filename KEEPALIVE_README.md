# Keep-Alive Script для Render.com

Этот скрипт предотвращает "засыпание" бесплатных сервисов на Render.com, отправляя периодические запросы.

## Сервисы

Скрипт пингует следующие URL:
- https://agroscore-app.onrender.com/
- https://agrocredit-api.onrender.com/

## Установка

1. Установите зависимости:
```bash
pip install -r requirements-keepalive.txt
```

## Запуск

### Локально (для тестирования)
```bash
python keep_alive.py
```

### В фоновом режиме (Linux/Mac)
```bash
nohup python keep_alive.py > keepalive.log 2>&1 &
```

### В фоновом режиме (Windows)
```powershell
Start-Process python -ArgumentList "keep_alive.py" -WindowStyle Hidden
```

## Как это работает

- Скрипт отправляет GET запрос на каждый URL каждые **14 минут**
- Render.com усыпляет бесплатные сервисы после 15 минут неактивности
- Логи показывают статус каждого запроса
- При ошибке скрипт автоматически повторяет попытку

## Остановка скрипта

Нажмите `Ctrl+C` для остановки скрипта.

## Альтернативные решения

### 1. Использование cron-job.org
Вместо запуска скрипта локально, можно использовать бесплатный сервис [cron-job.org](https://cron-job.org):
- Создайте аккаунт
- Добавьте оба URL
- Настройте выполнение каждые 14 минут

### 2. Использование UptimeRobot
[UptimeRobot](https://uptimerobot.com) - бесплатный сервис мониторинга:
- Бесплатно до 50 мониторов
- Проверка каждые 5 минут
- Email уведомления при сбоях

### 3. GitHub Actions
Можно настроить GitHub Actions для периодического пинга (требует публичный репозиторий).

## Примечания

- Скрипт должен работать постоянно для эффективности
- Рекомендуется запускать на сервере или использовать облачные решения
- Для продакшена лучше использовать платные планы Render.com без автоматического засыпания
